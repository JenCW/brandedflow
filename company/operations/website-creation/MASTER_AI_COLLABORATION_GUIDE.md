## üèóÔ∏è Website Creation Workflow

### Phase 1: Planning & Strategy

#### 1.1 Client Discovery
**Ask AI:**
```
"Create a comprehensive client discovery questionnaire for [client type] 
that covers business goals, target audience, competitors, brand positioning, 
and success metrics."
```

**Best Practices:**
- Always start with business goals, not design preferences
- Understand the customer journey before building
- Identify conversion goals (leads, sales, signups)
- Research competitors thoroughly

#### 1.2 Site Architecture
**Ask AI:**
```
"Based on this client profile [paste profile], create a site architecture 
with page hierarchy, user journey maps, and conversion funnels. Include 
wireframe descriptions for each page."
```

**Key Elements:**
- Homepage (hero, services, testimonials, CTA)
- Service/Product pages
- About page
- Portfolio/Case studies
- Contact/Lead capture
- Blog (if SEO-focused)
- Landing pages (for campaigns)

#### 1.3 Design Direction
**Ask AI:**
```
"Create a design brief for [client] that includes:
- Color palette (primary, secondary, accent)
- Typography (headings, body, display)
- Visual style (modern, luxury, bold, minimal)
- Image style guidelines
- Component library description"
```

**Design Principles:**
- **Mobile-first**: Design for mobile, enhance for desktop
- **Accessibility**: WCAG 2.1 AA minimum
- **Performance**: Fast loading = better rankings
- **Brand consistency**: Every element reinforces brand

### Phase 2: Development

#### 2.1 Template Selection
**Ask AI:**
```
"Review available templates and recommend the best match for [client] 
based on [design brief]. Consider: design style, functionality needs, 
SEO structure, and conversion optimization."
```

**Template Considerations:**
- Matches brand aesthetic
- Has conversion-optimized layouts
- SEO-friendly structure
- Mobile responsive
- Fast loading

#### 2.2 Content Implementation
**Ask AI:**
```
"Write all website copy for [client] including:
- Homepage hero headline and subheadline
- Service descriptions (SEO-optimized)
- About page narrative
- CTA button copy (varied, action-oriented)
- Meta descriptions for all pages
- Alt text for images"
```

**Copy Principles:**
- **Clear value proposition**: What do you do? Why choose you?
- **Benefit-focused**: Features tell, benefits sell
- **Action-oriented CTAs**: "Get Started" not "Learn More"
- **SEO-optimized**: Natural keyword integration
- **Scannable**: Headlines, bullets, short paragraphs

#### 2.3 Technical Implementation
**Ask AI:**
```
"Build the [page name] page with:
- Semantic HTML5 structure
- CSS with [design system]
- Responsive breakpoints (mobile, tablet, desktop)
- Accessibility attributes (ARIA labels, alt text)
- Schema.org markup
- Fast-loading optimizations"
```

**Technical Standards:**
- Semantic HTML (header, nav, main, section, article, footer)
- CSS Grid/Flexbox for layouts
- Mobile-first responsive design
- Image optimization (WebP, lazy loading)
- Fast page load (< 3 seconds)
- Clean, maintainable code

#### 3.1 SEO Foundation
**Ask AI:**
```

## üîç SEO & SERP Optimization

### On-Page SEO

#### Title Tags
**Format:** `Primary Keyword | Secondary Keyword | Brand Name`
**Length:** 50-60 characters
**Best Practice:** Front-load primary keyword

**Ask AI:**
```
"Create optimized title tags for all pages of [client] website. 
Include primary and secondary keywords naturally. Keep under 60 characters."
```

#### Meta Descriptions
**Format:** Compelling description with CTA
**Length:** 150-155 characters
**Best Practice:** Include primary keyword, create desire, add CTA

**Ask AI:**
```
"Write compelling meta descriptions for all pages that:
- Include primary keyword naturally
- Create desire/curiosity
- Include a subtle CTA
- Stay under 155 characters"
```

#### Header Structure
**Hierarchy:**
- H1: One per page, primary keyword
- H2: Main sections, secondary keywords
- H3: Subsections, long-tail keywords
- H4-H6: Supporting content

**Ask AI:**
```
"Review the header structure on [page] and optimize for SEO:
- Ensure one H1 with primary keyword
- Use H2s for main sections with secondary keywords
- Use H3s for subsections with long-tail keywords
- Maintain logical hierarchy"
```

### Technical SEO

#### Schema.org Markup
**Types to Use:**
- LocalBusiness (for local SEO)
- Service (for service pages)
- Organization (for about page)
- Review (for testimonials)
- FAQPage (for FAQ sections)

**Ask AI:**
```
"Add comprehensive Schema.org JSON-LD markup to [page] including:
- [Schema type] with all required properties
- Address, phone, email (NAP consistency)
- Service offerings
- Aggregate ratings
- Opening hours"
```

#### Site Speed
**Targets:**
- First Contentful Paint: < 1.8s
- Largest Contentful Paint: < 2.5s
- Time to Interactive: < 3.8s

**Ask AI:**
```
"Optimize [page] for speed by:
- Compressing images (WebP format)
- Minifying CSS/JS
- Implementing lazy loading
- Removing unused code
- Optimizing fonts (subset, display: swap)
- Adding caching headers"
```

#### Mobile Optimization
**Checklist:**
- Responsive design (all breakpoints)
- Touch-friendly buttons (44x44px minimum)
- Readable text (16px minimum)
- Fast mobile load time
- No horizontal scrolling

**Ask AI:**
```
"Audit [page] for mobile optimization:
- Check responsive breakpoints
- Verify touch target sizes
- Test text readability
- Check mobile load speed
- Ensure no layout issues"
```

## üîë Keyword Research & Implementation

### Keyword Research Process

#### 1. Primary Keywords
**Types:**
- Branded (company name)
- Service (what you do)
- Location-based (where you serve)
- Problem-solving (what problems you solve)

**Ask AI:**
```
"Research primary keywords for [client] in [industry] serving [location]:
- Service-based keywords
- Location-based keywords
- Problem-solving keywords
- Competitor keywords
- Search volume estimates
- Competition level"
```

#### 2. Long-Tail Keywords
**Characteristics:**
- 3-5 words
- Lower competition
- Higher conversion intent
- Specific queries

**Ask AI:**
```
"Generate long-tail keywords for [client] including:
- Question-based keywords (how, what, why)
- Comparison keywords (best, top, vs)
- Problem-solving keywords
- Location + service combinations
- Estimated search volume and difficulty"
```

#### 3. Semantic Keywords
**Related terms:**
- Synonyms
- Related concepts
- Industry terms
- LSI (Latent Semantic Indexing) keywords

**Ask AI:**
```
"Find semantic and LSI keywords related to [primary keyword]:
- Synonyms and variations
- Related industry terms
- Conceptually related keywords
- Co-occurring terms
- Topic clusters"
```

### Keyword Implementation

#### Content Optimization
**Best Practices:**
- Natural integration (no stuffing)
- Primary keyword in H1
- Secondary keywords in H2s
- Long-tail in body content
- Semantic keywords throughout

**Ask AI:**
```
"Optimize [page] content for these keywords:
- Primary: [keyword]
- Secondary: [keyword], [keyword]
- Long-tail: [keyword], [keyword]
Ensure natural integration, proper density, and semantic relevance."
```

#### URL Structure
**Format:** `domain.com/primary-keyword` or `domain.com/category/primary-keyword`

**Ask AI:**
```
"Create SEO-friendly URLs for all pages:
- Include primary keyword
- Keep URLs short and readable
- Use hyphens, not underscores
- Avoid unnecessary words
- Maintain logical hierarchy"
```

#### Internal Linking
**Strategy:**
- Link from high-authority pages
- Use keyword-rich anchor text
- Create topic clusters
- Link to relevant content

**Ask AI:**
```
"Create an internal linking strategy for [site]:
- Identify high-authority pages
- Map keyword-rich anchor text
- Create topic clusters
- Link related content
- Ensure logical flow"

## ‚ùå Common Mistakes to Avoid

### Design Mistakes

1. **Too Many Fonts**
   - **Fix:** Limit to 2-3 font families
   - **Ask AI:** "Reduce font families to 2-3 maximum while maintaining hierarchy"

2. **Poor Contrast**
   - **Fix:** Ensure WCAG AA compliance (4.5:1 for text)
   - **Ask AI:** "Check and fix all contrast ratios to meet WCAG AA standards"

3. **Cluttered Layouts**
   - **Fix:** Use white space, clear hierarchy
   - **Ask AI:** "Simplify [section] layout with better spacing and hierarchy"

### Content Mistakes

1. **Feature-Focused Instead of Benefit-Focused**
   - **Fix:** Rewrite to focus on benefits
   - **Ask AI:** "Rewrite [section] to focus on benefits, not just features"

2. **Weak CTAs**
   - **Fix:** Use action-oriented, specific language
   - **Ask AI:** "Rewrite all CTAs to be more action-oriented and specific"

3. **Keyword Stuffing**
   - **Fix:** Natural keyword integration
   - **Ask AI:** "Review [content] and remove keyword stuffing while maintaining SEO value"

### SEO Mistakes

1. **Missing Alt Text**
   - **Fix:** Add descriptive alt text to all images
   - **Ask AI:** "Add descriptive, keyword-rich alt text to all images on [page]"

2. **Poor Header Hierarchy**
   - **Fix:** One H1, logical H2-H6 structure
   - **Ask AI:** "Fix header hierarchy on [page] - ensure one H1 and logical structure"

3. **Missing Schema Markup**
   - **Fix:** Add appropriate schema markup
   - **Ask AI:** "Add comprehensive schema.org markup to [page]"

### Technical Mistakes

1. **Slow Load Times**
   - **Fix:** Optimize images, minify code, lazy load
   - **Ask AI:** "Optimize [page] for speed - compress images, minify code, implement lazy loading"

2. **Not Mobile-First**
   - **Fix:** Design for mobile, enhance for desktop
   - **Ask AI:** "Audit and fix mobile responsiveness on [page]"

3. **Accessibility Issues**
   - **Fix:** WCAG compliance, keyboard navigation, screen readers
   - **Ask AI:** "Fix accessibility issues on [page] - WCAG compliance, keyboard nav, screen readers"
